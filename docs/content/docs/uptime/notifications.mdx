---
title: Notifications
description: Get notified when monitors go down or recover
---

Configure where incident notifications are sent. Eesee supports email, Slack, Discord, and SMS. Channels are configured per organization and can be filtered to specific monitors.

## Setting up a channel

1. Go to **Settings → Notifications**
2. Scroll to the **Uptime Alert Channels** section
3. Click the channel type you want to add
4. Fill in the details and click **Create Channel**

The channel is active immediately.

## Channel types

### Email

Sends an HTML email when a monitor goes down and when it recovers. The email includes the monitor name, URL, error reason, and a link to the incident in your dashboard.

**Setup:** Enter any email address. Confirmation is not required — the address receives notifications immediately.

### Slack

Sends formatted Slack messages via an incoming webhook. Messages include the monitor name, status change (down or recovered), error details, and a direct link to the incident.

**Setup:**
1. In Slack, go to **Settings → Apps → Incoming Webhooks**
2. Click **Add New Webhook to Workspace**
3. Choose the channel and copy the webhook URL
4. Paste the URL into Eesee

### Discord

Sends Discord embeds via a channel webhook.

**Setup:**
1. In your Discord server, right-click the channel → **Edit Channel → Integrations → Webhooks**
2. Click **New Webhook**, copy the URL
3. Paste the URL into Eesee

### SMS

Sends a text message to a phone number. The message includes the monitor name and status.

**Setup:** Enter a phone number in E.164 format — country code first, no spaces or dashes (e.g. `+14155552671`).

SMS is available on Pro and Scale plans.

## Filtering by monitor

Each channel can be configured to only fire for specific monitors. By default, a channel receives notifications from all monitors in your organization.

To restrict a channel to specific monitors:

1. Click **Edit** on the channel
2. Under **Monitors**, select the monitors you want this channel to fire for
3. Save

Leave the monitor list empty to receive notifications from all monitors.

## Enabling and disabling channels

Use the toggle in the channel row to temporarily disable a channel without deleting it. Disabled channels receive no notifications.

## Testing a channel

Click **Test** on any channel to send a test notification immediately. Use this to verify the configuration is correct before relying on it in production.

## Cooldown

There is no per-channel cooldown for uptime notifications — you receive one notification when a monitor goes down and one when it recovers, regardless of how long the outage lasts.
