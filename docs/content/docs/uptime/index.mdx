Eesee Metrics includes built-in uptime monitoring. Add HTTP or TCP monitors, get notified the moment something goes down, track incident history, and share a public status page with your users.

## What's included

<Cards>
  <Card title="Monitors" href="/docs/uptime/monitors">
    Create HTTP and TCP monitors with custom validation rules, response time thresholds, and multi-region checks.
  </Card>
  <Card title="Incidents" href="/docs/uptime/incidents">
    Every downtime event is logged as an incident. Acknowledge, investigate, and resolve incidents with full timeline history.
  </Card>
  <Card title="Notifications" href="/docs/uptime/notifications">
    Get notified via email, Slack, Discord, or SMS when a monitor goes down or recovers.
  </Card>
</Cards>

## Two views: per-site and organization-wide

Monitors can be scoped to a specific site or viewed across your entire organization.

- **Site view** — go to any site → **Monitoring** in the sidebar to see monitors tagged to that site
- **Organization view** — go to **Uptime** from the main navigation to see all monitors across all sites

Both views show the same data. The site view is filtered to the current site; the org view shows everything.
